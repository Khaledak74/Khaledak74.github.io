// LSASS Credential Dumping Detection & Response

Cyber Kill Chain Simulation | MITRE ATT&CK T1003.001

Author: Motaw3 | Date: July 2026

1. Executive Summary

This lab documents a complete red team vs. blue team simulation focused on detecting LSASS (Local Security Authority Subsystem Service) credential dumping attacks using Splunk SIEM. The scenario follows the Cyber Kill Chain framework across all 7 phases, from initial reconnaissance to credential exfiltration and analysis.

The attacker successfully performed four distinct credential dumping techniques:

  • Task Manager (GUI) — Create dump file on lsass.exe
  • ProcDump (Sysinternals) — Command-line memory dump
  • Mimikatz — Industry-standard credential extraction tool
  • LOLBins (rundll32 + comsvcs.dll) — Living Off The Land stealth technique

2. Lab Environment & Architecture

The environment consists of a Kali Linux attacker, a Windows Server 2022 victim (Domain Controller), and an Ubuntu-based Splunk SIEM.

Role OS IP Address Purpose
Attacker Kali Linux 2026.1 192.168.1.5 Offensive operations
Victim Windows Server 2022 192.168.1.7 Domain Controller / Target

3. Data Sources

Key data sources include Windows Sysmon logs, specifically monitoring Event IDs 1 (Process Create), 7 (Image Load), 10 (Process Access), and 11 (File Create).

4. The 4 LSASS Credential Dumping Techniques

Technique 1: Task Manager (GUI Method)

An attacker with GUI access can right-click lsass.exe in Task Manager and select "Create dump file." Detection: Sysmon EID 10 (taskmgr.exe accessing lsass.exe).

Technique 2: ProcDump (Sysinternals)

Using the legitimate ProcDump tool with the -ma flag. Detection: Sysmon EID 1 launched with -ma lsass command line.

Technique 3: Mimikatz

The most well-known credential extraction tool. Detection: Sysmon EID 1 execution with command line containing sekurlsa or privilege::debug.

Technique 4: LOLBins — rundll32 + comsvcs.dll

Using rundll32.exe to call MiniDumpW in comsvcs.dll. This is extremely stealthy as both are signed Microsoft binaries.

5. Splunk Detection & Dashboard

A unified dashboard was created to correlate attacker commands with victim events. Key SPL queries focus on monitoring unusual access to the LSASS process memory.

index=main host=winserver22 sourcetype=xmlwineventlog EventCode=10 TargetImage="*lsass.exe*"

6. Conclusion

Detection coverage was successful across all four techniques using Sysmon. Behavioral detection remains critical for identifying LOLBins and legitimate tool abuse.
